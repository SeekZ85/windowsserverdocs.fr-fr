# <a name="optimizing-windows-10-version-1803-for-a-virtual-desktop-infrastructure-vdi-role"></a>Optimisation de Windows 10, version 1803, pour un rôle d’Infrastructure VDI (Virtual Desktop)

Cet article vous permet de choisir les paramètres pour Windows 10, version 1803 (build 17134) qui doit aboutir à des performances optimales dans un environnement virtualisé Desktop Infrastructure (VDI). Tous les paramètres dans ce guide sont *recommandations à prendre en considération* et se trouvent dans aucune exigence de façon.

Dans un environnement VDI, notamment pour optimiser les performances de Windows 10 est à minimiser se redessine application graphique, les activités en arrière-plan qui n’ont aucun intérêt majeur à l’environnement VDI, et de réduire généralement des processus en cours d’exécution pour le strict minimum. Un secondaire vise à réduire l’utilisation de l’espace disque dans l’image de base pour le strict minimum. Avec VDI implémentations, la plus petite possible de base ou taille de l’image « gold », peut légèrement réduire l’utilisation de la mémoire sur l’hyperviseur, ainsi que d’une réduction de la petite global des opérations réseau requis pour fournir l’image du bureau au consommateur.

> [!NOTE]  
> Paramètres recommandés ici peuvent être appliqués à une autre installation de Windows 10, version 1803, y compris ceux sur physiques ou d’autres appareils virtuels. Aucune recommandation dans cette rubrique ne doit affecter la prise en charge de Windows 10, version 1803.

> [!TIP]  
> Un script qui implémente les optimisations présentées dans cette rubrique, ainsi que d’un fichier d’exportation de stratégie de groupe que vous pouvez importer avec **LGPO.exe**--est disponible à l’adresse [TheVDIGuys](https://github.com//TheVDIGuys) sur GitHub.

## <a name="vdi-optimization-principles"></a>Principes de l’optimisation d’infrastructure VDI

Un environnement VDI présente une session de bureau complet, y compris les applications, à un utilisateur de l’ordinateur sur un réseau. Les environnements VDI utilisent généralement une image de système d’exploitation, qui devient alors la base pour les postes de travail présentés par la suite aux utilisateurs pour le travail. Il existe des variations des implémentations d’infrastructure VDI telles que « persistante », « non persistant » et « session Bureau ». Le type persistant préserve les modifications du système d’exploitation de bureau VDI à partir d’une session à l’autre. Le type non persistant ne conserve pas les modifications apportées au système d’exploitation de bureau VDI à partir d’une session à l’autre. À l’utilisateur ce bureau est légèrement différent de ceux de tout autre périphérique virtuel ou physique, hormis qu’il est accessible via un réseau.

Les paramètres d’optimisation a lieu sur un appareil de référence. Une machine virtuelle est un endroit idéal pour créer l’image, car vous pouvez enregistrer l’état, vérifiez les points de contrôle et les sauvegardes peuvent être effectuées et autres tâches utiles. Commencez par installer le système d’exploitation par défaut sur la machine virtuelle de base et puis Optimisez la machine virtuelle de base pour une utilisation de l’infrastructure VDI en supprimer les applications inutiles, l’installation des mises à jour de Windows, l’installation d’autres mises à jour, suppression des fichiers temporaires, d’appliquer des paramètres, etc.

Il existe d’autres types d’infrastructure VDI telles que des Services Bureau persistants et à distance (RDS). Une description approfondie de ces technologies est en dehors de l’étendue de cette rubrique, qui se concentre sur le Windows de base des paramètres d’image avec une référence à d’autres facteurs dans l’environnement telles que de l’optimisation de l’hôte.

### <a name="persistent-vdi"></a>VDI persistant

VDI persistant est au niveau de base, une machine virtuelle qui enregistre l’état du système d’exploitation entre les redémarrages. Autres couches logicielles de la solution VDI fournissent l’accès de simple et transparente des utilisateurs sur leurs ordinateurs virtuels affectés, souvent avec une solution d’authentification unique.

Il existe plusieurs implémentations différentes de la solution VDI persistant :

-   Traditionnels de machines virtuelles, où la machine virtuelle possède son propre fichier de disque virtuel, démarre normalement, enregistre les modifications d’une session à l’autre et est en fait simplement une machine virtuelle normale. La différence est la façon dont l’utilisateur accède à cette machine virtuelle. Il peut y avoir un portail web de l’utilisateur se connecte qui dirige automatiquement l’utilisateur à leur ou plus affecté des machines virtuelles d’infrastructure VDI.

-   Image persistante machine virtuelle, avec des disques virtuels personnels. Dans ce type d’implémentation est une image de base/gold un ou plusieurs serveurs hôtes. Une machine virtuelle est créée, et un ou plusieurs disques virtuels sont créés et affectés à ce disque pour le stockage persistant.

    -   Lorsque la machine virtuelle est démarrée, une copie de l’image de base est en lecture dans la mémoire de la machine virtuelle. En même temps, un disque virtuel persistant affecté à la machine virtuelle, avec les modifications de système d’exploitation précédent fusionnées via un processus complexe.

    -   Modifications telles que les écritures du journal des événements, les écritures du journal, etc. sont redirigées vers le disque virtuel en lecture/écriture affecté à cette machine virtuelle.

    -   Dans ce cas, système d’exploitation et la maintenance de l’application peuvent fonctionner normalement, à l’aide de logiciels de maintenance traditionnels tels que Windows Server Update Services ou d’autres technologies de gestion.

### <a name="non-persistent-vdi"></a>VDI non persistant

Lorsqu’une implémentation de VDI non persistant est basée sur une base ou d’une image « gold », les optimisations sont effectuées principalement dans l’image de base, puis via des paramètres locaux et les stratégies locales.

Avec l’infrastructure VDI non persistant basé sur image, l’image de base est en lecture seule. Lorsqu’une VM VDI non persistant est démarrée, une copie de l’image de base est diffusé en continu à la machine virtuelle. Activité qui se produit lors du démarrage et par la suite, jusqu’au redémarrage suivant est redirigée vers un emplacement temporaire. Généralement, les utilisateurs sont fournies les emplacements réseau pour stocker leurs données. Dans certains cas, le profil utilisateur est fusionné avec la machine virtuelle standard pour permettre aux utilisateurs leurs paramètres.

Maintenance d’un aspect important de la solution VDI non persitent qui repose sur une seule image. Mises à jour du système d’exploitation sont généralement remis une fois par mois.
Avec VDI basé sur image, il existe un ensemble de processus à effectuer afin d’obtenir les mises à jour de l’image :

-   Sur un hôte donné, toutes les machines virtuelles sur cet ordinateur hôte qui sont dérivés de l’image de base doit être arrêté ou désactivé. Cela signifie que les utilisateurs sont redirigés vers les autres machines virtuelles.

-   L’image de base est ensuite ouvert et démarré. Toutes les activités de maintenance sont ensuite effectuées, telles que les mises à jour du système d’exploitation, les mises à jour de .NET, les mises à jour de l’application, etc.

-   Tous les paramètres qui doivent être appliquées sont appliquées à ce stade.

-   Autres tâches de maintenance est effectuée à ce stade.

-   L’image de base est ensuite arrêté.

-   L’image de base est sealed et défini pour revenir en arrière en production.

-   Les utilisateurs sont autorisés pour vous reconnecter.

> [!NOTE]  
> Windows 10 effectue automatiquement, un ensemble de tâches de maintenance sur une base périodique. Il est une tâche planifiée qui est définie pour s’exécuter à 3 h 00, heure locale quotidiennement par défaut. Cette tâche planifiée exécute une liste de tâches, y compris le nettoyage de la mise à jour de Windows. Vous pouvez afficher toutes les catégories de maintenance qui se produisent automatiquement avec cette commande PowerShell :

`Get-ScheduledTask | ? {$_.Settings.MaintenanceSettings}`



Un des défis liés à l’infrastructure VDI non persistant est que lorsqu’un utilisateur se déconnecte, presque toutes les activités de système d’exploitation sont ignorée. Le profil utilisateur et ou état peut être enregistré, mais la machine virtuelle elle-même ignore presque toutes les modifications qui ont été apportées depuis le dernier démarrage. Par conséquent, les optimisations destinées à un ordinateur Windows qui enregistre l’état d’une session à l’autre ne sont pas applicables.

Selon l’architecture de VDI VM, les choses like prérécupération et SuperFetch ne vont pas à l’aide d’une session à l’autre, comme toutes les optimisations sont ignorées sur machine virtuelle redémarrer. L’indexation peut être une perte partielle de ressources, comme ce seraient les optimisations de disque comme une opération de défragmentation traditionnelle.

### <a name="to-sysprep-or-not-sysprep"></a>Sysprep ou pas Sysprep

Windows 10 dispose d’une fonctionnalité intégrée appelée le [outil de préparation système](https://docs.microsoft.com/windows-hardware/manufacture/desktop/sysprep--system-preparation--overview), (souvent abbreviiated à « Sysprep »). L’outil Sysprep sert à préparer une image personnalisée de Windows 10 pour la duplication. Le processus Sysprep garantit que le système d’exploitation qui en résulte est correctement unique pour exécuter en production.

Il existe des raisons pour et contre l’exécution de Sysprep. Dans le cas d’infrastructure VDI, vous souhaiterez peut-être la possibilité de personnaliser le profil utilisateur par défaut qui doit être utilisé en tant que le modèle de profil pour les utilisateurs suivants qui se connectent à l’aide de cette image. Vous pouvez avoir des applications que vous voulez installé, mais également en mesure de contrôler les paramètres par application.

L’alternative consiste à utiliser une norme. ISO pour installer à partir, en utilisant un fichier de réponses d’installation sans assistance et une séquence de tâches pour installer des applications ou de supprimer des applications. Vous pouvez également utiliser une séquence de tâches pour définir des paramètres de stratégie locale dans l’image, par exemple à l’aide de la [Local groupe stratégie objet utilitaire (LGPO)](https://blogs.technet.microsoft.com/secguide/2016/01/21/lgpo-exe-local-group-policy-object-utility-v1-0/) outil.

### <a name="universal-windows-platform-app-cleanup"></a>Nettoyage d’application Universal Windows Platform

Un des objectifs d’une image de l’infrastructure VDI doit être aussi réduite que possible. Une pour réduire la taille de l’image consiste à supprimer des applications UWP qui ne seront pas utilisées dans l’environnement. Avec les applications UWP, il existe les fichiers d’application principale, également connu sous le nom de la charge utile. Il existe une petite quantité de données stockées dans chaque profil utilisateur pour les paramètres spécifiques aux applications. Il existe également une petite quantité de données dans le profil de tous les utilisateurs.

Connectivité et minutage sont tout ce que lorsqu’il s’agit de nettoyage d’application UWP. Si vous déployez votre image de base sur un appareil sans connectivité réseau, Windows 10 ne peut pas se connecter à la Microsoft Store et télécharger des applications et essayez de les installer lorsque vous essayez de les désinstaller.

Si vous modifiez votre base. WIM que vous utilisez pour installer Windows 10 et supprimer les applications UWP à partir de la. WIM avant d’installer, les applications ne seront pas installées pour commencer et vos heures de création de profil doivent être plus courts. Plus loin dans cette section, vous trouverez plus d’informations sur la suppression des applications UWP à partir de votre installation. Fichier WIM.

Une bonne stratégie pour l’infrastructure VDI consiste à configurer les applications souhaitées dans l’image de base, puis limiter ou bloquer l’accès pour le Microsoft Store par la suite. Applications de Store sont régulièrement mis à jour en arrière-plan sur les ordinateurs normaux. Les applications UWP peuvent être mis à jour au cours de la fenêtre de maintenance lorsque les autres mises à jour sont appliquées. 

#### <a name="delete-the-payload-of-uwp-apps"></a>Supprimer la charge utile des applications UWP

Les applications UWP qui ne sont pas nécessaires sont toujours en consommant une petite quantité d’espace disque du système de fichiers. Pour les applications qui ne seront jamais nécessaire, la charge utile des applications UWP indésirables peut être supprimée à partir de l’image de base à l’aide des commandes PowerShell.

En fait, si vous supprimez ceux à partir de l’installation. Fichier WIM en utilisant les liens prévues plus tard cette section, vous devez être en mesure de démarrer à partir du début avec une liste très léger des applications UWP.

Exécutez la commande suivante pour énumérer les applications UWP configurées à partir d’un système d’exploitation Windows 10 en cours d’exécution, comme dans cet exemple tronquée de sortie à partir de PowerShell :

```powershell

    Get-AppxProvisionedPackage -Online 
    
    DisplayName  : Microsoft.3DBuilder
    Version      : 13.0.10349.0  
    Architecture : neutral
    ResourceId   : \~ 
    PackageName  : Microsoft.3DBuilder_13.0.10349.0_neutral_\~_8wekyb3d8bbwe 
    Regions      : 
    ...
```


Les applications UWP qui sont configurées sur un système peuvent être supprimées pendant l’installation de système d’exploitation en tant que partie d’une séquence de tâches, ou plus tard, une fois le système d’exploitation est installé. Cela peut être la méthode recommandée, car elle rend le processus de création ou la gestion d’une image modulaire. Une fois que vous développez les scripts, si quelque chose change dans une build ultérieure, vous modifiez un script existant plutôt que répéter le processus à partir de zéro. Voici quelques liens vers des informations sur cette rubrique :

[Suppression d’applications de zone dans Windows 10 pendant une séquence de tâches](https://blogs.technet.microsoft.com/mniehaus/2015/11/11/removing-windows-10-in-box-apps-during-a-task-sequence/)

[Suppression des applications intégrées à partir du fichier WIM de Windows 10 avec Powershell - Version 1.3](https://gallery.technet.microsoft.com/Removing-Built-in-apps-65dc387b)

[Windows 10 1607 : Maintenir des applications à partir de venir lors du déploiement de la mise à jour de fonctionnalité](https://blogs.technet.microsoft.com/mniehaus/2016/08/23/windows-10-1607-keeping-apps-from-coming-back-when-deploying-the-feature-update/)

Exécutez ensuite la [Remove-AppxProvisionedPackage](https://docs.microsoft.com/powershell/module/dism/remove-appxprovisionedpackage?view=win10-ps) commande PowerShell pour supprimer des charges utiles d’application UWP :

`Remove-AppxProvisionedPackage -Online -PackageName MyAppxPackage`

Chaque application UWP doit être évaluée pour la mise en application dans chaque environnement. Vous souhaitez installer une installation par défaut de Windows 10, version 1803, puis notez les applications qui sont en cours d’exécution et consommation de mémoire. Par exemple, vous souhaiterez peut-être envisager de supprimer des applications qui démarrent automatiquement, ou qui affichent automatiquement des informations sur le menu Démarrer, tels que la météo et les actualités, et cela pourrait s’avérer utile dans votre environnement.

L’une des applications UWP « boîte de réception » appelées Photos, a un paramètre appelé par défaut **afficher une notification lorsque de nouveaux albums sont disponibles**.  L’application de Photos peut utiliser environ 145 Mo de mémoire ; utilisation privée spécifiquement définie mémoire, même si ce n’est pas utilisé.  Modification de la **afficher une notification lorsque de nouveaux albums sont disponibles** paramètre pour tous les utilisateurs n’est pas pratique à ce moment, par conséquent, la recommandation pour supprimer l’application de Photos si elle n’est pas nécessaire ou souhaité.

### <a name="clean-up-optional-features"></a>Nettoyer les fonctionnalités facultatives

#### <a name="managing-optional-features-with-powershell"></a>La gestion des fonctionnalités facultatives avec PowerShell

 Pour énumérer les fonctionnalités de Windows actuellement installée, exécutez cette commande PowerShell :

`Get-WindowsOptionalFeature -Online`


Vous pouvez activer ou désactiver une fonctionnalité facultative de Windows spécifique, comme dans cet exemple :

`Enable-WindowsOptionalFeature -Online -FeatureName "DirectPlay" -All`

Pour plus d’informations, consultez [Windows 10 : La gestion des fonctionnalités facultatives avec PowerShell](https://social.technet.microsoft.com/wiki/contents/articles/39386.windows-10-managing-optional-features-with-powershell.aspx).

#### <a name="enable-or-disable-windows-features-by-using-dism"></a>Activer ou désactiver des fonctionnalités de Windows à l’aide de DISM

Vous pouvez utiliser l’intégrée **Dism.exe** outil pour énumérer et contrôler les fonctionnalités facultatives de Windows. Vous pouvez configurer un script Dism.exe pour s’exécuter pendant une séquence de tâches qui installe le système d’exploitation.

### <a name="tuning-windows-10-network-performance-by-using-registry-settings"></a>Réglage des performances de réseau de Windows 10 à l’aide des paramètres du Registre

Ceci est particulièrement important dans les environnements où l’infrastructure VDI ou l’ordinateur physique a une charge de travail qui est principalement le réseau en fonction. Les paramètres dans cette section les performances polarisation de favoriser la mise en réseau, en configurant supplémentaires de mise en mémoire tampon et de mise en cache des choses telles que les entrées d’annuaire et ainsi de suite.

Notez que certains paramètres dans cette section sont *basée sur le Registre uniquement* et doit être incorporé dans l’image de base avant le déploiement de l’image pour la production.

Les paramètres suivants sont documentés dans le [indication de réglage de performances Windows Server 2016](https://docs.microsoft.com/windows-server/administration/performance-tuning/) plus d’informations, publiées sur Microsoft.com par le groupe de produit Windows.

#### <a name="disablebandwidththrottling"></a>DisableBandwidthThrottling

HKLM\\System\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters\\DisableBandwidthThrottling

S’applique à Windows 10. La valeur par défaut est **0**. Par défaut, le redirecteur SMB limite le débit sur les connexions réseau à latence élevée, dans certains cas afin d’éviter des délais d’attente liés au réseau. La valeur du Registre **1** désactive cette limitation, l’activation de débit de transfert de fichier plus élevé sur les connexions réseau à latence élevée, vous devez donc envisager ce paramètre.

#### <a name="fileinfocacheentriesmax"></a>FileInfoCacheEntriesMax

HKLM\\System\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters\\FileInfoCacheEntriesMax

S’applique à Windows 10. La valeur par défaut est **64**, avec une plage valide de 1 à 65 536. Cette valeur permet de déterminer la quantité de métadonnées de fichier pouvant être mises en cache par le client. Augmenter la valeur, vous pouvez réduire le trafic réseau et améliore les performances lorsque de nombreux fichiers sont accessibles. Essayez d’augmenter cette valeur à **1024**.

#### <a name="directorycacheentriesmax"></a>DirectoryCacheEntriesMax

HKLM\\System\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters\\DirectoryCacheEntriesMax

S’applique à Windows 10. La valeur par défaut est **16**, avec une plage valide de 1 à 4 096. Cette valeur permet de déterminer la quantité d’informations de répertoire pouvant être mises en cache par le client. Augmenter la valeur peut réduire le trafic réseau et améliorer les performances lorsque des répertoires volumineux sont consultés. Envisagez d’augmenter cette valeur à **1024**.

#### <a name="filenotfoundcacheentriesmax"></a>FileNotFoundCacheEntriesMax

HKLM\\System\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters\\FileNotFoundCacheEntriesMax

S’applique à Windows 10. La valeur par défaut est **128**, avec une plage valide de 1 à 65 536. Cette valeur permet de déterminer la quantité d’informations de nom de fichier pouvant être mises en cache par le client. Augmenter la valeur, vous pouvez réduire le trafic réseau et améliore les performances lorsque de nombreux noms de fichiers sont accessibles. Envisagez d’augmenter cette valeur à **2048**.

#### <a name="dormantfilelimit"></a>DormantFileLimit

HKLM\\System\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters\\DormantFileLimit

S’applique à Windows 10. La valeur par défaut est **1023**. Ce paramètre spécifie le nombre maximal de fichiers devant être laissés ouverts sur une ressource partagée une fois que l’application a fermé le fichier. Lorsque vous connectent plusieurs milliers de clients aux serveurs SMB, réduisez cette valeur pour **256**.

Vous pouvez configurer plusieurs de ces paramètres SMB à l’aide de la [Set-SmbClientConfiguration](https://docs.microsoft.com/powershell/module/smbshare/set-smbclientconfiguration) et [Set-SmbServerConfiguration](https://docs.microsoft.com/powershell/module/smbshare/set-smbserverconfiguration) applets de commande Windows PowerShell. Vous pouvez configurer les paramètres de Registre uniquement à l’aide de Windows PowerShell, comme dans l’exemple suivant :

`Set-ItemProperty -Path "HKLM:\\SYSTEM\\CurrentControlSet\\Services\\LanmanWorkstation\\Parameters" RequireSecuritySignature -Value 0 -Force`

### <a name="disk-cleanup-including-using-the-disk-cleanup-wizard"></a>Nettoyage de disque (y compris à l’aide de l’Assistant de nettoyage de disque)

Nettoyage de disque peut être particulièrement utile avec les implémentations d’infrastructure VDI image maître. Une fois l’image principale est préparé, mis à jour et configuré, les dernières tâches à effectuer est le nettoyage de disque. L’Assistant de nettoyage de disque intégré à Windows peut aider à nettoyer la plupart des zones potentielles d’économies d’espace disque.

> [!NOTE]  
> L’Assistant de nettoyage de disque n’est plus en cours de développement. Windows utilisera les autres méthodes pour fournir des fonctions de nettoyage de disque.



Voici les suggestions pour diverses tâches de nettoyage de disque. Vous devez les tester avant d’implémenter une d'entre elles :

1.  Après avoir appliqué les mises à jour, exécutez l’Assistant de nettoyage de disque (avec élévation de privilèges). Inclure les catégories **optimisation de la distribution** et **nettoyage de mise à jour Windows**. Vous pouvez automatiser ce processus avec **Cleanmgr.exe** avec la **/sageset : 11** option. Cette option définit les valeurs de Registre qui peuvent être utilisés ultérieurement pour automatiser le nettoyage de disque, à l’aide de toutes les options disponibles dans l’Assistant de nettoyage de disque.

    1.  Sur un machine virtuelle, de test à partir d’une nouvelle installation, en cours d’exécution **/sageset : 11 Cleanmgr.exe** révèle qu’il existe seulement deux options de nettoyage de disque automatique est activées par défaut :

    - Fichiers programme téléchargés

    - Fichiers Internet temporaires

    2.  Si vous définissez des options supplémentaires, ou toutes les options, ces options sont enregistrées dans le Registre, en fonction de la valeur d’index fournie dans la commande précédente ( **/sageset : 11 Cleanmgr.exe**). Dans cet exemple, nous utilisons la valeur *11* comme notre index, pour une procédure de nettoyage automatique du disque ultérieures.

    3.  Après l’exécution **/sageset : 11 Cleanmgr.exe** vous verrez un nombre de catégories d’options de nettoyage de disque. Vous pouvez sélectionner toutes les options, puis sélectionnez **OK**. Vous remarquerez que l’Assistant de nettoyage de disque disparaît. Toutefois, les paramètres que vous avez sélectionné sont enregistrés dans le Registre et peuvent être appelées en exécutant **Cleanmgr.exe /SAGERUN:11**.

2.  Nettoyer le stockage de cliché instantané de Volume, si une est en cours d’utilisation. Pour ce faire, exécutez les commandes suivantes dans une invite de commandes avec élévation de privilèges :

    - **vssadmin list shadows**

    - **Shadowstorage de liste vssadmin**

        Si la sortie de ces commandes est *aucun élément trouvé qui satisfont à la requête.* , alors il n’existe aucun stockage VSS en cours d’utilisation.

3.  Fichiers temporaires de nettoyage et de journaux. À partir d’une invite de commandes avec élévation de privilèges, exécutez les commandes suivantes :

    - **DEL C:\\\*.tmp /s**

    - **DEL C:\\Windows\\Temp\\.**

    - **Del % temp%\\.**

4.  Supprimer tous les profils inutilisés sur le système avec cette commande :

    **WMIC chemin win32_UserProfile où LocalPath = "c:\\\\utilisateurs\\\\\<utilisateur\>« supprimer**
