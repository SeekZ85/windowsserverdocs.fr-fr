# <a name="use-ram-that-provides-error-correction"></a>Utiliser la mémoire vive qui fournit la correction des erreurs

>S'applique à : Windows Server 2016

Pour plus d'informations sur les meilleures pratiques et les analyses, consultez [Best Practices Analyzer](https://go.microsoft.com/fwlink/?LinkId=122786).  
  
|Propriété|Détails|  
|-|-|  
|**Système d'exploitation**|Windows Server 2016|  
|**Produit/fonctionnalité**|Hyper-V|  
|**Niveau de gravité**|Erreur|  
|**Catégorie**|Configuration|  
  
Dans les sections suivantes, italique indique le texte de l’interface utilisateur qui apparaît dans l’outil Best Practices Analyzer pour ce problème.  
  
## <a name="issue"></a>Problème  
  
*La mémoire vive utilisée sur cet ordinateur n'est pas correction d’erreur (ECC) de RAM.*  
  
## <a name="impact"></a>Impact  
  
*Microsoft ne prend pas en charge les Windows Server 2016 sur les ordinateurs sans RAM de correction d’erreurs.*  
  
## <a name="resolution"></a>Résolution  
  
*Vérifiez que le serveur est répertorié dans le catalogue Windows Server et qualifié pour Hyper-V.*  
  
Pour vérifier si le serveur est répertorié, consultez le [catalogue Windows Server](https://www.windowsservercatalog.com/).
